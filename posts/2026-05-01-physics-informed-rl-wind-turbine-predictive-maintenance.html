物理驅動強化學習：下一代風力機預測性維護的融合路徑

2026-05-01 • 劉瑞弘教授團隊 • WindAILab / 預測性維護 / Physics-Informed RL / SCADA

WindAILab Physics-Informed RL 預測性維護 SCADA 資料

風力發電機組的維護決策是一個兼具物理複雜性與經濟不確定性的難題。齒輪箱磨損、軸承疲勞、葉片積冰——這些故障都有其物理演進規律，但真實風場中的隨機風況、複雜海況（離岸風機）以及有限的維修窗口，使得「何時派人維修、何時讓機組繼續運轉」成為一個高度動態的最佳化問題。傳統的基於閾值的預警系統過於保守，容易誤報；純資料驅動的深度學習模型雖然精度高，但在物理上不可解釋、且面對分佈外資料時泛化性差。

2026 年初，一篇發表於 Energy and AI 的研究提出了一個新框架：物理驅動強化學習（Physics-Informed Reinforcement Learning, PI-RL），將 Weibull 風速分佈、多階段劣化模型與溫度-劣化雙向耦合直接嵌入 RL 智能體的獎勵函數設計，並在真實風場 SCADA 資料上取得了顯著成效。這個方向，與我們 DOF Lab WindAILab 的技術路線高度共鳴。

一、為什麼純資料驅動不夠：物理知識的不可或缺性

在 DOF Lab 過去兩年的 WindAILab 開發中，我們深刻體會到一件事：高 F1 分數並不等於可部署的維護系統。一個在訓練集上 F1=1.0 的 SCADA 故障分類器，在面對新安裝機型、季節性氣候偏移或設備老化後，準確率可能快速下滑。這背後的根本原因是資料分佈偏移（Distribution Shift）——而物理定律不會偏移。

以齒輪箱劣化為例：無論資料集如何變化，「負載增大 → 溫升加劇 → 潤滑油膜薄化 → 磨損加速」這條物理因果鏈始終成立。如果我們能把這條鏈編碼進模型，模型就擁有了超越訓練資料的外推能力。這正是 Physics-Informed Neural Networks（PINN）與 PI-RL 的核心賭注：用物理知識換取資料效率與泛化性。

二、PI-RL 框架的關鍵設計：獎勵函數的物理嵌入

PI-RL 在強化學習框架中的核心創新，在於如何設計一個同時反映經濟最佳化與物理可行性的獎勵函數（Reward Function）。傳統 RL 維護模型的獎勵設計通常僅考慮維修成本與停機損失的線性組合，忽略了物理約束——例如，在風機劣化程度尚淺時派出維修船（離岸風場的主要成本項目），等同於浪費高昂的船班費用。

新框架的獎勵函數包含三個耦合項：

  • 劣化懲罰項：根據 Weibull 多階段劣化模型，當前狀態下劣化進展速度越快，懲罰越大，促使智能體提早介入。
  • 維修機會獎勵項：根據 Weibull 風速分佈估算未來 7 天的預期維修窗口（適合出海的天氣），將窗口稀缺性納入決策，避免「等到故障才發現明天有強風」。
  • 溫度-劣化耦合懲罰：雙向耦合項，考慮高溫不只加速劣化，劣化本身也會因摩擦增加而提升溫度，形成正回饋環。當系統進入此正回饋區域時，智能體獲得強烈懲罰，促使緊急維護。

💡 物理嵌入的實作要點

獎勵函數中的物理項並非靜態常數，而是基於實時 SCADA 資料動態估算的：

  • 溫度序列 → 劣化速率估算（多段 Arrhenius 方程擬合）
  • 振動頻譜 → 當前劣化階段判斷（Stage 1/2/3）
  • 歷史風速記錄 → Weibull 參數 MLE 更新（滾動視窗）
  • 油溫 + 負載 → 潤滑膜薄化風險指數計算

⚠️ 關鍵：物理量的計算需整合到 RL 環境（Gym Environment）的 step() 函數，而非後處理，才能實現真正的「物理驅動」而非「物理裝飾」。

三、與 WindAILab 的接軌：從故障分類走向決策智能化

WindAILab 目前的核心能力，是透過 25 個協同 AI 代理完成「SCADA 清洗 → 特徵工程 → ML 故障分類 → 自然語言報告生成」的完整診斷管線，並在 Phase 14 中完成了多代理狀態管理的系統性重構。下一個自然延伸方向，正是將診斷能力升級為維護決策能力。

PI-RL 框架提供了一條清晰的路徑：WindAILab 的故障分類輸出（故障類型、嚴重程度分級）可直接作為 RL 環境的狀態觀測向量的一部分。這意味著我們不需要從頭建構維護決策系統，而是在現有診斷管線頂部疊加一個決策層——故障診斷智能體的輸出成為決策智能體的輸入。

🏗️ WindAILab + PI-RL 整合架構構想

  • Layer 1（現有）：診斷層 — SCADA 清洗 → 特徵工程 → 故障分類（ML 多代理）→ 故障類型 + 嚴重程度分級
  • Layer 2（新增）：狀態估算層 — 故障輸出 + SCADA 時序 → 物理劣化模型（Weibull + 溫度耦合）→ 劣化階段向量
  • Layer 3（新增）：決策層 — 劣化向量 + 風場維修窗口預測 + 成本模型 → PI-RL 智能體 → 維護動作（立即維修 / 計畫維修 / 繼續監控）
  • Layer 4（現有）：報告層 — 診斷 + 決策 → WindGuard AI LLM → 自然語言維護建議報告（含 PDF 輸出）

這個四層架構最大的優勢是模組化可替換性：Layer 1 的分類器可以從 XGBoost 換成 PatchTST（我們在 Phase 12 已完成 Benchmark），Layer 3 的 RL 智能體可以從 PPO 換成 SAC，而整體系統結構不變。這也呼應了軟體工程中「關注點分離（Separation of Concerns）」的原則——每一層只需對自己的介面負責。

四、實務挑戰：離岸風場的資料稀缺性問題

PI-RL 聽起來很理想，但在台灣離岸風場的實際應用中，有一個根本性的挑戰難以迴避：故障樣本極端稀少。一座 100 MW 離岸風場，一年可能只有 3–5 次嚴重故障事件，這對任何需要大量失敗案例的 RL 訓練都是嚴峻考驗。

目前研究社群對此的主流解法有兩條路線：模擬器預訓練（Sim-to-Real Transfer），先在高保真度的風機劣化模擬環境中訓練智能體，再用少量真實資料微調；以及遷移學習（Transfer Learning），從資料豐富的陸上風場或同類型設備遷移已訓練的 RL 策略，再以台灣風場的少量資料做域適應。兩種方法都依賴物理模型的精確性——物理知識在此扮演了「資料增強」的角色，填補了真實故障樣本的空缺。

  • 建立齒輪箱溫度-劣化耦合模型的 Python 實作原型，基於 WindAILab 已有的 SCADA 特徵工程模組延伸。

  • 與 ECN（教育課程導航）研究的蒙地卡羅不確定性量化方法交叉應用：將維修窗口預測的不確定性納入 RL 獎勵函數的風險項。
