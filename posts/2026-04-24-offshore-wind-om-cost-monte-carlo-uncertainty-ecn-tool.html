從單點估算到機率分布：Monte Carlo 仿真如何重塑離岸風電 O&M 成本分析

2026-04-24 • 劉瑞弘教授團隊 • 離岸風電 / O&M 成本 / Monte Carlo / ECN O&M Tool

Monte Carlo 仿真 離岸風電 O&M LCOE 不確定性 ECN O&M Tool

一座 130 台風機的離岸風場，25 年運轉期間的 O&M（Operation & Maintenance，營運維護）成本，可能高達數十億新台幣。然而，傳統的試算表工具給出的往往只是一個數字——一個「預計總成本」，不帶任何誤差範圍，不說明最壞情況，也不量化關鍵假設的敏感度。

這種「點估算」（point estimate）的思維，在離岸風電高度不確定的現實環境中，是危險的過度自信。海象等待時間可能因一個颱風季而倍增；零件故障率隨老化非線性攀升；維修船的日租金受國際市場波動。任何一個假設偏離，都可能讓成本分析徹底失效。

DOF Lab 的 ECN O&M Tool 正在推進 Phase 4——以 Monte Carlo 仿真取代確定性計算，將 O&M 成本的輸出從「一個數字」變成「一條機率分布曲線」，讓決策者真正理解他們面對的是什麼樣的風險。

一、離岸風電 O&M 成本的不確定性來源

離岸風電的 O&M 成本分析，面對的是多層疊加的不確定性。理解這些不確定性的結構，是設計 Monte Carlo 仿真的前提。

氣象不確定性（Metocean Uncertainty）：離岸作業高度依賴海象條件。波高超過 1.5 公尺、風速超過 12 m/s，維修船便無法作業。ECN O&M Tool 使用 Weibull 分布對歷史海象資料進行參數擬合，但年際變異可能使「平均等待時間」偏差達 30% 以上。氣候變遷的加入，更使長期預測充滿系統性誤差。

設備可靠性不確定性（Reliability Uncertainty）：風機故障率遵循浴缸曲線（Bathtub Curve）——早期有磨合期故障，中期相對穩定，晚期因材料疲勞而加速劣化。ECN O&M Tool 的 VarFluct 模組已在 Phase 3 完成浴缸曲線生命週期建模，但曲線的精確形狀在設備老化過程中始終帶有統計不確定性。

成本結構不確定性（Cost Structure Uncertainty）：零件採購價格、維修船日租金、技術人員工資，受通膨率、國際供應鏈、市場競爭影響。25 年的通膨假設若偏差 1%，累積效果便可能造成數億元的估算誤差。

二、Monte Carlo 仿真的核心設計

Monte Carlo 方法的基本思想是：對所有帶有不確定性的輸入參數，賦予機率分布而非固定值；然後重複抽樣數千次，每次計算出一個 O&M 成本結果；最終用這些結果的統計分布，描述真實世界中可能出現的各種情境。

ECN O&M Tool Phase 4 的 Monte Carlo 模組包含四個關鍵元件：

📐 參數分布設定

每個不確定輸入可被指定為正態分布、三角分布、均勻分布或對數正態分布。海象等待時間以對數正態分布建模（對應極端天氣的長尾特性），零件成本以三角分布建模（具有明確的最小值、最可能值與最大值）。

🔄 仿真引擎

預設執行 10,000 次抽樣（可配置至 100,000 次）。採用 Latin Hypercube Sampling（LHS）取代純粹隨機抽樣，確保在較少的仿真次數下達到更均勻的參數空間覆蓋，有效降低收斂所需計算量。

📈 結果可視化

仿真完成後自動產生三種視圖：直方圖（Histogram）顯示成本分布形狀，累積分布函數（CDF）方便讀取任意百分位值，以及龍捲風圖（Tornado Chart）呈現各輸入變數對總成本變異的貢獻排序。

🎯 百分位卡片

儀表板自動計算 P10（樂觀情境）、P50（最可能情境）、P90（保守情境）三個關鍵百分位，配合均值與標準差，讓非技術背景的決策者也能快速掌握風險輪廓。

三、Tornado Chart：找出真正左右成本的關鍵變數

在 Monte Carlo 仿真的眾多輸出中，Tornado Chart（龍捲風圖）往往是最有決策價值的分析工具。它的邏輯是：逐一讓每個輸入變數在其 P10 到 P90 範圍內變動，觀察總成本的變動幅度，再將所有變數按影響力大小排序，整體形狀酷似龍捲風。

結果顯示：海象條件的不確定性遠超其他因素，其影響力甚至超過傳統上被認為最重要的「零件故障率」。這個發現對台灣離岸風電具有特別意義——台灣海峽的颱風頻率與北海、波羅的海截然不同，直接套用歐洲成熟市場的 O&M 成本模型，可能嚴重低估氣象等待時間的成本佔比。

🌊 台灣離岸風電的 ECN 應用特殊性

ECN O&M Tool 的原型基於荷蘭 ECN（現為 TNO）開發的 Excel 工具，主要針對北海環境設計。DOF Lab 的研究工作之一，是驗證並調整其參數設定，使其適用於台灣海峽環境：

  • 颱風季節性效應：6–10 月的作業限制明顯高於北海，需以季節性分布取代全年均勻假設
  • CTV vs. SOV 選擇：台灣現階段主要依賴 CTV（運維交通船），而非歐洲成熟市場的 SOV（服務運維母船）；兩者的成本結構與氣象作業限制截然不同
  • 本地化供應鏈：隨台灣離岸風電供應鏈本地化進展，維修成本的地域性差異需動態更新

四、Monte Carlo 結果如何連結 LCOE 與投資決策

Monte Carlo 仿真的最終價值，不在於製造更複雜的圖表，而在於讓財務決策建立在更誠實的基礎上。ECN O&M Tool 的 Phase 4 將仿真結果直接饋入 LCOE（平準化能源成本）計算模組，輸出「機率分布化的 LCOE」。

這意味著：投資者可以問「在 90% 的情境下，LCOE 是否低於電費保證收購價（FiT）？」而不是「預期 LCOE 是多少？」前者是風險管理思維，後者只是樂觀願景。P90 的 LCOE 若仍低於收購電價，這個風場才具有真正穩健的投資價值。

## ECN O&M Tool Phase 4 — Monte Carlo LCOE 計算示意 from ecn_tool.monte_carlo import MonteCarloEngine from ecn_tool.lcoe import LCOECalculator # 定義不確定性輸入 engine = MonteCarloEngine(n_simulations=10000, method="LHS") engine.add_param("weather_wait_factor", dist="lognormal", mu=1.0, sigma=0.25) engine.add_param("ctv_day_rate", dist="triangular", low=8000, mode=12000, high=18000) engine.add_param("major_component_fr", dist="normal", mu=0.015, sigma=0.004) engine.add_param("inflation_rate", dist="uniform", low=0.015, high=0.035) # 執行仿真 results = engine.run(cost_model=project_config) # 輸出百分位 print(f"P10 O&M Cost: NT$ {results.percentile(10)/1e8:.2f} 億") print(f"P50 O&M Cost: NT$ {results.percentile(50)/1e8:.2f} 億") print(f"P90 O&M Cost: NT$ {results.percentile(90)/1e8:.2f} 億") # 連結 LCOE lcoe_dist = LCOECalculator.from_monte_carlo(results, aep=annual_energy_production) print(f"P90 LCOE: {lcoe_dist.percentile(90):.4f} NT$/kWh")

這樣的分析框架，也讓運維策略的比較（Corrective Maintenance vs. Condition-Based Maintenance）從「哪個方案成本較低？」升級為「哪個方案的 P90 成本風險較可接受？」——這才是成熟風險管理的正確問法。

🎯 ECN O&M Tool Phase 4 下一步

  • Monte Carlo 驗證：以 K13 案例的歷史實際 O&M 成本資料，反向驗證仿真輸出的機率分布是否合理包含歷史真實值
  • 情景比較升級：現有情景比較模組將整合 Monte Carlo 輸出，讓不同運維策略以 P50 vs. P90 帶狀圖呈現，而非單點對比
  • VarFluct 浴缸曲線耦合：Phase 3 完成的 VarFluct 生命週期故障率模型將作為 Monte Carlo 的可靠性輸入分布來源，實現從元件老化曲線到系統風險分布的完整連結
  • 學術驗證：計畫投稿至 Journal of Physics: Conference Series（Wind Energy 專題），以 Monte Carlo 方法重現並延伸 ECN 原始工具的驗證案例

風險不是要被消除的東西——它是要被量化、被理解、然後被定價的東西。對離岸風電的投資決策者而言，知道「最糟糕的情況有多糟、發生的機率有多大」，遠比知道「預期是多少」更有價值。

ECN O&M Tool 的 Monte Carlo 之路，是 DOF Lab 在風能數位化研究上的一個縮影：不只是讓計算更快，而是讓分析更誠實。
